A) HTML Structure - top

This template is a fixed layout with a single column that contains possibility to add multiple columns. All of the information within the main content area is nested within a div with an id of "content". The general template structure is the same throughout the template. Here is the general structure.

	<div id="root">
		<header id="top">
			<p id="logo"><a href="./" accesskey="h">Logo</a></p>
			
		</header>                  
		<section id="content">
			
		</section>
		<footer id="footer">
		
		</footer>
	</div>

Please note that HTML structure is adjustable so you are free to copy/switch/mix elements as you assure that proper HTML structure is kept. If you would like to create a content that is divided into two columns, please add following markup:

	<div class="double">
		<div>
			Content for first column
		</div>
		<div>
			Content for second column
		</div>
	</div>

Also, you can add additional classes along with "double", such as "a" to set spacing between items, "inv" to invert columns or "middle" to align elements to center. All of the uses can be investigated on template pages.

If you would like to edit the color, font, or style of any elements in one of these columns, you would do the following:

	a {
		color: #someColor;
	}

If you find that your new style is not overriding, it is most likely because of a specificity problem. Scroll down in your CSS file and make sure that there isn't a similar style that has more weight.

I.E.

	#root a {
		color: #someColor;
	}

So, to ensure that your new styles are applied, make sure that they carry enough "weight" and that there isn't a style lower in the CSS file that is being applied after yours.

B) CSS Files and Structure - top

We're using one CSS file in this template. There is no a generic reset file because we go one step ahead and predifine all of the HTML structure, so we can work round browser incompatibilities. This file also contains some general styling, such as anchor tag colors, font-sizes, etc. Keep in mind, that these values might be overridden somewhere else in the file.

The file is separated into sections using:

	/*! Layout --------- */
	Core layout structure, usually defined with IDs.

	/*! Columns --------- */
	If you'd like to use two columns view, here's where all the CSS is defined.
	
	/*! Modules --------- */
	Generic modules that are repeatable thorough the website.
	
	/*! Headers --------- */
	Definitions for headings: h1, h2, h3, h4, h5, h6.
	
	/*! Content --------- */
	Definitions for generic content.
	
	/*! Links --------- */
	Definitions for links.
	
	/*! Lists --------- */
	Definitions for lists, mostly ol and ul
	
	/*! Media --------- */
	Definitions for media items, such as images or videos.
	
	/*! Forms --------- */
	Complex CSS standarizer for various form elements and definitions for custom form tags.
	
	/*! Tables --------- */
	Definitions for tables.
	
	/*! Helpers --------- */
	Generic helpers, such as transitions, hiding elements or setting consistent margins.
	
	/*! Warnings --------- */
	Warnings for users that have old versions of Interner Explorer browser or don't support JS (needs to be enabled).
	
	/*! Miscellaneous --------- */
	Misc stuff, more standarizing.

	/*! Fonts --------- */
	Custom fonts listed along with this project.
	
	/*! Animations --------- */
	All of the CSS3 animations.
	
	/*! Icons --------- */
	All of the icons uniquely created for this project.
	
	/*! Typicons --------- */
	Default "typicons" set.

	/* Responsive --------- */
	CSS for responsive versions, divided into separate media queries for each resolution.

If you would like to edit a specific section of the site, simply find the appropriate label in the CSS file, and then scroll down until you find the appropriate style that needs to be edited.

C) JavaScript - top

This template imports three Javascript files.

  1. jQuery
  2. Scripts.js
  3. Custom.js
  4. Mobile.js
  1. jQuery is - a Javascript library that greatly reduces the amount of code that you must write.
  2. Scripts.js - contains all libraries used in the project, each one is compressed to ensure the fastest loading.
  3. Customs.js - is a place where you can find all of the code created uniquely for this project. It is devided into few sections:
    • links - very basic script that helps to avoid "target="_blank" attribute which is not recommended from SEO perspective.
    • mails - putting email address directly can be insecure, we recommend to put it in "name//domain/com" format, so this script can turn it into readable email with slightly better security.
    • forms - all of the custom JS code to handle special behaviour of forms.
    • heights - sometimes, we need to play with heights to equalize them.
    • date - if you don't like to update they year each January, here's the script that will do this for you.
    • responsive - here's the place where we play a bit with JS for mobile devices.
    • ie.css - some slight tweaks just for IE.
    • loading - if you like our CSS/JS preloaders, here's where they are defined.
    • animations - although all of the animations are handled by CSS, we use a JS to define when they should run.
    • miscellaneous - all of the generic stuff
  4. Mobile.js - contains some awesome features that helps standarizing stuff on various mobile devices
